# Безопасность

## Секреты

- `GEMINI_API_KEY` и `E2B_API_KEY` используются только backend-процессом.
- Секреты запрещено помещать в переменные `NEXT_PUBLIC_*`.
- `.env`, `.env.local` и `.refs` исключены из Git.
- Ключи из публичной переписки считаются скомпрометированными и должны быть заменены.

## Защита от prompt-injection

Приложение использует несколько независимых уровней защиты.

1. Содержимое датасета не вставляется в исходный prompt.
2. Модель получает только путь к файлу, имя, размер и пользовательскую аналитическую задачу.
3. System instruction явно определяет ячейки, названия колонок и tool output как недоверенные данные.
4. Инструкции из датасета не могут отменить системные правила.
5. Пользовательский контекст очищается от управляющих символов и ограничен 2000 символами.
6. Финальный отчет принимается только через строго проверенную Zod-схему.

## Ограничение интерпретатора

- E2B создается с `allowInternetAccess: false`.
- В песочницу не передаются API-ключи.
- Разрешен только Python tool, shell tool отсутствует.
- Код проверяется перед запуском.
- Блокируются системные, сетевые и процессные модули.
- Блокируются `eval`, `exec`, `__import__`, shell magic, чтение окружения и прямой `open`.
- Запуск ограничен 45 секундами.
- Песочница уничтожается в `finally`.

## Ограничение данных и ответа

- максимальный файл: 10 МБ;
- форматы: CSV и XLSX;
- содержимое файла проверяется на соответствие расширению;
- один клиент может запустить не более 6 анализов за 10 минут;
- максимум 8 агентных ходов;
- максимум 4 графика;
- stdout и результаты сокращаются до безопасного размера;
- модель должна выполнить минимум два успешных Python-вызова;
- полный датасет и персональные строки запрещено печатать в prompt.

## CORS

Backend принимает запросы только от origin из `ALLOWED_ORIGINS`. Для production укажите точные адреса без путей:

```env
ALLOWED_ORIGINS=https://linsivs.github.io
```

## Реакция на утечку

1. Выпустить новые ключи Gemini и E2B.
2. Заменить секреты в Render.
3. Отозвать старые ключи.
4. Проверить usage/quota в кабинетах провайдеров.
5. Убедиться, что ключей нет в Git history.
