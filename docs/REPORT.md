# Отчет по заданию 3

## Название

«Разбор»: веб-продукт с LLM-аналитикой датасетов.

## Цель

Создать интерфейс, в котором пользователь загружает CSV или Excel и получает аналитический отчет, сформированный LLM-агентом через программный API и реальный интерпретатор кода.

## Реализованный сценарий

Пользователь загружает файл и может добавить контекст: например, попросить сравнить регионы, найти причины падения показателя или проверить влияние скидок. Backend загружает файл в одноразовую E2B-песочницу. Gemini самостоятельно вызывает Python-инструмент, исследует структуру, рассчитывает метрики, строит графики и только после проверки результатов формирует отчет.

После отчета пользователь может открыть полноэкранный чат по тому же датасету.
Backend сохраняет E2B-сессию на 15 минут. Для каждого вопроса Gemini обязана
снова вызвать `run_python`, получает фактический вывод и только затем вызывает
`submit_answer`. Ответ содержит доказательства, графики и раскрываемый журнал
новых вычислений.

## Доказательство агентности

Приложение не передает модели заранее рассчитанные характеристики датасета. Модель получает путь к файлу и сама:

1. генерирует Python-код;
2. вызывает `run_python`;
3. получает stdout, ошибки и изображения;
4. корректирует последующие шаги;
5. вызывает `submit_report`.

Интерфейс показывает журнал каждого запуска: цель, код, время и фактический вывод.

В follow-up чате действует то же доказательство агентности: сервер не передает
модели готовую статистику, а предоставляет инструмент Python и путь к
сохраненному файлу.

## Защита от prompt-injection

- данные таблицы считаются недоверенными;
- содержимое файла не копируется в исходный prompt;
- инструкции из ячеек должны игнорироваться;
- E2B работает без интернета и без API-ключей;
- опасные Python-конструкции блокируются до исполнения;
- пользовательская инструкция ограничена по размеру;
- итоговый отчет проверяется строгой схемой.
- вопросы чата и история диалога считаются недоверенными данными;
- ответ чата принимается только после успешного Python-вызова.

## Тестирование

Команда `pnpm check` выполняет:

- ESLint;
- TypeScript typecheck;
- 21 автоматический тест;
- production build Next.js.

Интерфейс дополнительно проверен в браузере на ширине 1280 px и 390 px.
Проверены демонстрационный отчет, открытие полноэкранного чата, отправка
вопроса, desktop/mobile раскладка и отсутствие ошибок в консоли.

## Архитектурное ограничение

GitHub Pages поддерживает только статический frontend. API-ключи нельзя помещать в браузерный bundle, поэтому backend разворачивается отдельно на Render, а Pages вызывает его по HTTPS.
